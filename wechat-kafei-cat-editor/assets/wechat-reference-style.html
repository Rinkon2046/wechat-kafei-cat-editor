AGENT HARNESS · 图文整理

为什么你的 Agent 项目
总是难以落地

很多 Agent 项目不是死在模型不够聪明，而是死在业务正反馈到来之前，团队被迫手搓 Agent Harness 这一层基础设施。

模型决定上限，Harness 决定兑现率，业务 Know-how 决定最终价值。

很多团队以为自己在做 Agent，其实是在重复造基础设施：上下文管理、工具调用、错误恢复、权限、安全、沙箱、可观测性、部署和成本控制。

结果是，真正懂业务的人，被迫把时间花在和业务差异很弱的工程问题上。

01

先把三层拆开

要理解 Agent 为什么难落地，先把 LLM、Agent Harness、Agent 这三层拆清楚。很多失败都来自把这三层混在一起。

LLM 是大脑。LLM 只能输入文字、输出文字。它本身不能读文件、不能运行代码、不能等待，也不能天然知道工具执行后的后果。

Agent Harness 是身体和神经系统。Harness 把模型的意图变成现实动作：读文件、调工具、跑测试、处理失败、管理上下文、做权限和安全控制。

Agent 是有职业的人。真正的 Agent = LLM + Harness + 某个业务场景里的知识、工具、流程和判断标准。

同一个模型、同一种 Harness，可以支撑销售、客服、交易、研发等不同 Agent。最后决定价值的，是它是否懂具体职业里的成功标准和工作流。

02

为什么 Claude Code 行，你的 Agent 不行

同样是 Claude，Claude Code 的效果好，不是因为单纯"Prompt 更好"，而是它有生产级 Harness。差距主要在这几处：

上下文管理：知道当前文件、最近改动、git 状态、依赖和终端报错，而不是把整个目录粗暴塞进 prompt。

工具边界：编辑文件不是整文件覆盖，而是结构化 diff、补丁应用、失败回滚。

错误恢复：测试失败、编译报错、lint 警告会自动回填给模型，让它继续修。

模型路由：不同任务可能交给不同强度的模型，而不是一把梭。

隐性 Prompt 工程：system prompt、工具描述、few-shot、停止条件都经过长期打磨。

这些细节叠在一起，就是"能用"和"不能用"的差距。

03

Agent 项目的真正卡点

很多业务团队的核心能力，本来是懂客户、懂行业、懂流程、懂成功标准。但现实里，他们大量时间都花在了 Harness 层。

交易 Agent 团队，本该研究策略、仓位、风控、市场状态，却在接行情、下单 API、异常回滚、监控稳定性。

营销 Agent 团队，本该研究用户分层、渠道转化、素材策略、campaign 节奏，却在处理工具失败、CRM 权限、A/B 数据回填、上下文截断。

业务团队真正值钱的是业务理解，但他们被迫同时做业务和基础设施。

让 SaaS 产品团队从零搭 Harness，就像让电商团队自己造数据库内核。也许能造出来，但市场窗口可能已经错过。

04

Harness 平台为什么必然出现

Web 早期，每个团队都要自己处理服务器、HTTP、数据库、机柜。后来 AWS、Vercel、云数据库等基础设施平台出现，应用团队才真正把精力放回业务。

Agent 现在正处在"Web 还没有 AWS"的阶段。Harness 平台的价值，是把工具调用、状态管理、可观测性、错误恢复、安全权限这些东西产品化，让业务团队只需要表达自己的业务思路。

把 Harness 当作水电煤，而不是护城河。

客户不会因为你"自研 Agent 框架"而多付钱。客户只会因为你更懂他们的业务、更能解决他们的问题而留下来。

05

未来 AI 行业的三层格局

第一层：模型层。OpenAI、Anthropic、Google、DeepSeek 这一类公司，资本密集、人才密集、算力密集，最终会高度集中。

第二层：Harness 平台层。它可能成为 AI 时代的 AWS，门槛是工程深度、生态广度和客户成功能力的组合。

第三层：Agent 应用层。会出现大量垂直应用公司，护城河在业务、客户和工作流理解。

最危险的位置：既不够 infra、又不够垂直的"半吊子 Agent 产品"，通用得不够通用，专业得不够专业，最后会被两头挤压。

06

给团队的落地判断清单

1. 我们团队真正擅长的是业务，还是 Agent 基础设施？

2. 这个项目的关键风险，是模型能力不够，还是 Harness 兑现率不够？

3. 我们现在 80% 的时间，是在做客户价值，还是在补工具调用、权限、上下文和监控？

4. 哪些基础设施应该买、租、用平台，哪些业务逻辑必须自己沉淀？

5. 如果把 Harness 层交出去，我们能不能更快拿到业务正反馈？

07

三句话转述版

① Agent 不是"模型 + Prompt"，中间还有一整层 Harness。

② 多数 Agent 项目落不了地，不是因为团队不懂业务，而是被迫先造基础设施。

③ 未来业务团队应该把 Harness 当水电煤，把精力放回客户、场景和工作流。
